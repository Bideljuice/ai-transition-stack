# Post Refiner & Quality Control

## When to Use This Skill
When Tam has a draft post and wants to pressure-test it
before publishing. Always run all passes in order.
Never skip a pass. Never deliver until the post passes
all checks or 10 passes are complete.

## CRITICAL
- Never deliver the post without running all 4 core checks
- Show which pass you're on and what failed — 
  don't rewrite silently
- Never add length — refinement only removes or sharpens
- Never change Tam's core idea or angle — 
  only improve how it's expressed
- If post passes all checks before pass 10 — 
  stop and deliver immediately
- If post fails after pass 10 — recommend starting fresh,
  do not keep rewriting

## Steps

Step 1: Receive draft
Tam pastes any draft post.
Identify format — LinkedIn, Twitter thread, 
single tweet, article, email.

Step 2: Run Pass 1 — AI Writing Check
Check against references/refinement-criteria.md Pass 1.
List every fail found.
Rewrite to fix all fails.
Show Tam: "Pass 1 complete. Fixed: [list what changed]"

Step 3: Run Pass 2 — Inspiration Check
Check against references/refinement-criteria.md Pass 2.
If no line removes an excuse or reframes something — 
identify the weakest line and rewrite it to do this.
Show Tam: "Pass 2 complete. Fixed: [list what changed]"

Step 4: Run Pass 3 — Voice Check
Check against references/refinement-criteria.md Pass 3.
Ask: could this have been written by anyone?
If yes — identify which lines and rewrite them as Tam.
Show Tam: "Pass 3 complete. Fixed: [list what changed]"

Step 5: Run Pass 4 — Density Check
Check against references/refinement-criteria.md Pass 4.
Cut or rewrite any sentence that doesn't earn its place.
Show Tam: "Pass 4 complete. Fixed: [list what changed]"

Step 6: Run Pass 5 — CTA Check
Check against references/refinement-criteria.md Pass 5.
If CTA is generic — rewrite it.
Show Tam: "Pass 5 complete. Fixed: [list what changed]"

Step 7: Passes 6-10 — Line by Line
Go through every line.
For each: "Does this earn its place?"
Cut or rewrite anything that doesn't.
Re-run all checks after each pass.
Stop the moment everything passes.

Step 8: Deliver
Present the final post only — clean, no explanation.
Below it show: "Passed on pass [X]. Changes made: [list]"
Ask: "Ready to publish?"

## Important
- Each pass builds on the last — don't undo previous fixes
- Refinement = sharper, shorter, more Tam
- Never longer, never softer, never more balanced
- Voice reference: my-writing-style skill

## Note
- Tam knows what's wrong when something feels off —
  if he flags something specific, address that first
  before running standard passes
- Morgan Housel style throughout —
  short sentences, one idea per paragraph,
  grounded in story or analogy
